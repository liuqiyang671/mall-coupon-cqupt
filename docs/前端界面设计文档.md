## 1. 项目概述

### 1.1 项目背景

邮惠券商城是由重庆邮电大学学生团队开发的优惠券管理与商城系统，基于微服务架构构建，为平台管理人员、入驻商家和普通消费者提供完整的优惠券生命周期管理能力。系统名称"邮惠券"取自"邮电大学"的"邮"与"优惠"的"惠"，体现项目的高校背景与优惠属性。

### 1.2 目标用户群体

| 用户角色 | 角色描述 | 核心诉求 |
|---------|---------|---------|
| **平台管理员** | 系统运营团队 | 全局数据监控、商家审核管理、系统配置运维 |
| **商家** | 入驻商户运营人员 | 商品管理、优惠券创建与发放、经营数据统计 |
| **普通用户** | CQUPT 在校学生 | 浏览商品、领取优惠券、购物车、下单结算 |

### 1.3 核心功能模块

```
邮惠券商城
├── B端管理后台（平台管理员 + 商家）
│   ├── 工作台仪表盘
│   ├── 商品管理（CRUD / 分类 / 属性 / 图片）
│   ├── 优惠券模板管理（创建 / 编辑 / 上下架 / 增发）
│   ├── 推送任务管理（批量发券 / 定时发送 / 进度跟踪）
│   └── 个人信息管理
│
└── C端用户界面（普通用户）
    ├── 首页（Banner / 分类导航 / 热门推荐 / 限时抢券）
    ├── 分类页（三级分类 / 筛选 / 排序）
    ├── 商品详情页（图片轮播 / 规格选择 / 优惠券关联）
    ├── 领券中心（可领券列表 / 同步领券 / MQ高峰领券）
    ├── 我的优惠券（按状态筛选 / 核销 / 退款）
    ├── 购物车（增删改 / 选中 / 价格计算 / 税费）
    ├── 结算页（用券计算 / 确认下单）
    ├── 预约提醒（创建 / 取消 / 到期通知）
    └── 个人中心（信息编辑 / 密码修改 / 订单记录）
```

## 2. 设计原则

### 2.1 一致性原则

| 维度 | 要求 |
|------|------|
| **视觉一致性** | 全站统一的色彩、字体、圆角、阴影规范，避免同一功能在不同页面呈现不同样式 |
| **交互一致性** | 相同操作使用相同的交互模式（如：删除操作统一使用二次确认弹窗） |
| **术语一致性** | 全站统一业务术语（如：优惠券/商品/结算单，不混用"券/卡/票"等同义词） |
| **布局一致性** | 同角色页面保持相同的导航结构和内容区域布局 |

### 2.2 可用性原则

| 原则 | 实践 |
|------|------|
| **可见性** | 操作入口明显可见，核心功能不超过 3 次点击可达 |
| **反馈性** | 每个操作都有即时反馈（Loading / Toast / 状态变更） |
| **容错性** | 危险操作二次确认，表单实时校验，错误信息明确可操作 |
| **效率性** | 高频操作提供快捷方式（Enter 提交、键盘导航、批量操作） |
| **可学习性** | 遵循用户已有认知模型，界面元素含义直观 |

### 2.3 可访问性原则

| 标准 | 要求 |
|------|------|
| **色彩对比度** | 文本与背景对比度 ≥ 4.5:1（WCAG 2.1 AA 级） |
| **焦点可见** | 所有可交互元素支持键盘 Tab 导航，焦点环清晰可见 |
| **触摸目标** | 移动端最小触摸区域 44×44px |
| **语义化** | 使用语义化 HTML 标签，ARIA 属性标注 |
| **动画可控** | 尊重 `prefers-reduced-motion` 系统设置 |

### 2.4 品牌特色原则

| 要求 | 说明 |
|------|------|
| **高校属性** | 融入重庆邮电大学视觉元素（校徽色、校园印象），体现学生项目特色 |
| **优惠属性** | 优惠券为核心视觉元素，红色系为主色调传达"优惠/折扣"心智 |
| **专业品质** | 虽为学生项目，但界面品质对标商业级电商系统 |

## 3. 色彩方案

### 3.1 品牌主色

邮惠券商城的品牌主色取自重庆邮电大学校徽的红色调，同时传达"优惠/折扣"的消费心理暗示。

| 色彩名称 | 色值 | 用途 | 示意 |
|---------|------|------|------|
| **邮惠红（主色）** | `#E1251B` | 品牌标识、主按钮、链接、选中态、优惠券金额 | ██████ |
| **邮惠红深（悬停）** | `#C81623` | 主色悬停态、按下态 | ██████ |
| **邮惠红浅（背景）** | `#FFF1F0` | 选中背景、标签背景、高亮行 | ██████ |

> **设计说明**：主色 `#E1251B` 为暖红色调，既呼应重邮校徽色系，又符合电商行业"促销红"的用户认知。与京东 `#F10215` 色值接近但不相同，确保品牌独立性。

### 3.2 辅助色

## 4. 排版系统

### 4.1 字体选择

| 用途 | 字体栈 | 说明 |
|------|--------|------|
| **主字体** | `"PingFang SC", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif` | 中文优先苹方/雅黑，英文回退 Helvetica |
| **等宽字体** | `"SF Mono", "Fira Code", "Consolas", monospace` | 代码/金额数字展示 |
| **数字字体** | `"DIN Alternate", "Roboto Mono", "SF Mono", monospace` | 优惠券金额、价格等大号数字 |

### 4.2 字号层级

借鉴京东的字号层级体系，建立从页面标题到辅助文字的完整层级：

| 层级 | 字号 | 字重 | 行高 | 用途 | 示例 |
|------|------|------|------|------|------|
| **H1** | 24px | 800 | 1.3 | 页面主标题 | 优惠券模板管理 |
| **H2** | 20px | 700 | 1.4 | 区域标题 | 基本信息 |
| **H3** | 16px | 600 | 1.5 | 卡片标题/表单分组 | 领取规则 |
| **Body** | 14px | 400 | 1.6 | 正文/表单标签/列表内容 | 商品名称 |
| **Caption** | 13px | 400 | 1.5 | 辅助说明/只读信息 | 创建时间 |
| **Small** | 12px | 400 | 1.5 | 标签/徽章/脚注 | 已下架 |

### 4.3 特殊数字排版

| 场景 | 字号 | 字重 | 说明 |
|------|------|------|------|
| **优惠券金额** | 46px | 900 | 领券中心/优惠券卡片上的大号金额 |
| **价格** | 20px | 700 | 商品价格、结算金额 |
| **原价** | 12px | 400 | 删除线样式 |
| **统计数字** | 32px | 800 | Dashboard KPI 数字 |

### 4.4 间距系统

采用 **8px 基准网格**，所有间距为 8 的倍数：

| 间距变量 | 值 | 用途 |
|---------|-----|------|
| `--space-xs` | 4px | 图标与文字间距、紧凑内边距 |
| `--space-sm` | 8px | 表单项间距、标签间距 |
| `--space-md` | 16px | 卡片内边距、列表项间距 |
| `--space-lg` | 24px | 区域间距、卡片外边距 |
| `--space-xl` | 32px | 页面区域分隔 |
| `--space-2xl` | 48px | 页面顶部/底部留白 |

## 7. 交互设计

### 7.1 核心交互流程

#### 领券交互流程

```
用户点击"立即领取"
    ↓
┌─── 按钮变为 Loading 态（旋转图标 + "领取中..."） ───┐
│                                                       │
│   ┌── 同步模式 ──┐     ┌── MQ 异步模式 ──┐          │
│   │ 请求后端接口  │     │ 请求后端接口     │          │
│   │ 等待响应      │     │ 立即返回"已提交" │          │
│   └──────────────┘     └─────────────────┘          │
│                                                       │
│   成功 → 按钮变"已领取"（灰色禁用）                   │
│        → Toast "领取成功！优惠券已放入您的账户"       │
│        → 卡片添加"已领取"标签                         │
│                                                       │
│   失败 → 按钮恢复"立即领取"                           │
│        → Toast 显示错误原因                           │
│        → 库存不足时按钮变"已抢光"                     │
└───────────────────────────────────────────────────────┘
```

#### 购物车交互流程

```
添加商品 → Toast "已加入购物车" + 购物车图标徽标 +1
    ↓
修改数量 → 输入框防抖 500ms → 乐观更新小计金额 → 请求后端
    ↓                                            ↓ 失败回滚
选中/取消 → 即时更新底部汇总金额（本地计算，无需请求）
    ↓
结算 → 跳转结算页（携带选中商品信息）
```

#### 商品搜索交互

```
输入关键词 → 300ms 防抖 → 显示搜索建议下拉
    ↓
选择建议 / 回车 → 跳转搜索结果页
    ↓
搜索结果页 → 筛选/排序 → 局部刷新列表
```

### 7.2 动画效果规范

| 动画场景 | 效果 | 时长 | 缓动函数 |

| **抽屉滑入** | translateX(100%→0) | 300ms | ease-out |

| 场景 | 反馈方式 | 持续时间 |
|------|---------|---------|
| **操作成功** | ElMessage.success（绿色图标 + 文字） | 2s |
| **操作失败** | ElMessage.error（红色图标 + 文字） | 3s |
| **警告提示** | ElMessage.warning（橙色图标 + 文字） | 3s |
| **网络错误** | ElMessage.error("网络异常，请稍后重试") | 常驻（可手动关闭） |
| **表单校验** | 输入框下方红色文字 + 输入框边框变红 | 实时 |
| **加载中** | v-loading 指令 / 骨架屏 / NProgress | 请求期间 |
| **空状态** | el-empty 组件 + 引导操作按钮 | 常驻 |
| **删除确认** | ElMessageBox.confirm 二次确认弹窗 | 等待用户操作 |
| **领券成功** | 卡片状态变更动画 + Toast | 2s |
| **库存预警** | 黄色标签"仅剩 N 件" | 常驻 |
