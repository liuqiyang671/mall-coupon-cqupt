# 全局技术规范-京东风格设计系统

## 1. 设计目标

本规范定义 OneCoupon 前端页面的统一视觉语言。用户端参考京东的优惠券、结算和促销转化设计；商家端参考京东商家后台的信息密度和操作效率，但降低营销化装饰。

## 2. 京东参考原则

| 参考点 | 用于本项目 | 注意事项 |
| --- | --- | --- |
| 红色主按钮 | 领取、确认使用、登录 | 商家端减少大面积红色 |
| 券卡片面额突出 | 领券中心、我的优惠券、结算抽屉 | 金额/折扣必须最醒目 |
| 不可用券置灰 | 我的优惠券、结算 | 必须给不可用原因 |
| 标签分组 | 我的优惠券、任务状态 | 保持短文本和明确状态 |
| 高密度表格 | 商家后台 | 操作列固定，危险操作二次确认 |

## 3. 设计 Token

| Token | 值 | 用途 |
| --- | --- | --- |
| `--color-brand` | `#e1251b` | 用户端主按钮、券金额 |
| `--color-brand-hover` | `#c81623` | 悬停 |
| `--color-success` | `#16a34a` | 成功状态 |
| `--color-warning` | `#f59e0b` | 待执行、提醒 |
| `--color-danger` | `#dc2626` | 删除、结束、失败 |
| `--color-text` | `#1f2937` | 主文本 |
| `--color-muted` | `#6b7280` | 辅助文本 |
| `--radius-card` | `8px` | 卡片 |

## 4. 组件规范

### 4.1 券卡片

| 区域 | 内容 | 规则 |
| --- | --- | --- |
| 面额区 | 金额/折扣 | 字号最大，品牌红 |
| 规则区 | 门槛、适用范围 | 最多两行 |
| 时间区 | 有效期 | 辅助色 |
| 操作区 | 领取/去使用/已领取 | 单主按钮 |

### 4.2 后台表格

- 筛选区在表格上方，按钮顺序为“查询、重置、创建”。
- 行操作不超过 3 个，更多操作放入下拉菜单。
- 危险操作使用二次确认，不直接执行。

## 5. 响应式规则

| 断点 | 用途 |
| --- | --- |
| `<768px` | 用户端移动布局，券卡片纵向 |
| `768-1199px` | 平板布局，表格可横向滚动 |
| `>=1200px` | 后台主工作区，固定侧边栏 |

## 6. 可访问性

- 主按钮对比度不低于 WCAG AA。
- 金额和状态不能只依赖颜色，需有文本标签。
- 弹窗打开后焦点进入弹窗，关闭后回到触发按钮。
