# 全局技术规范-测试与兼容性

## 1. 测试目标

保证前端页面在核心业务链路、异常场景、权限控制和浏览器兼容性方面可验证、可回归。

## 2. 单元测试

| 范围 | 工具 | 要求 |
| --- | --- | --- |
| 表单校验 | Vitest | 覆盖必填、格式、边界值 |
| 规则解析 | Vitest | 覆盖 `receiveRule`、`consumeRule` 异常 JSON |
| 金额计算 | Vitest | 覆盖满减、立减、折扣、精度 |
| 状态流转 | Vitest | 覆盖 loading/success/error |

## 3. 组件测试

| 组件 | 覆盖点 |
| --- | --- |
| 券卡片 | 可领、已领、已抢光、不可用 |
| 登录表单 | 校验、提交、错误展示 |
| 券选择抽屉 | 选中、取消、确认、不使用 |
| 后台表格 | 筛选、分页、行操作 |

## 6. 兼容性处理

- 日期显示统一使用格式化库，不依赖浏览器解析后端时间字符串。
- 金额统一使用 Decimal 或字符串计算。
- `position: sticky` 在不支持场景降级为普通布局。
- 移动端弹层避免固定高度，使用 `max-height` 和滚动容器。

## 7. 性能指标

| 指标 | 目标 |
| --- | --- |
| 用户端首屏 LCP | 小于 2.5s |
| 后台列表首屏 | 小于 1.5s |
| API 超时 | 10s |
| 交互反馈 | 点击后 100ms 内出现 Loading 或状态变化 |

## 8. 验收清单

- 每个模块页面有空状态、加载态、错误态。
- 所有危险操作有二次确认。
- 所有 API 错误可被用户理解。
- 所有 ID 按字符串处理。
- 所有模块文档与对应 API 文件一一对应。
