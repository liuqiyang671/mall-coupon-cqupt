# OneCoupon 前端开发技术文档体系

## 1. 文档目标

本文档体系用于指导 OneCoupon 优惠券系统前端页面实现，覆盖商家端、用户端、结算链路和跨模块工程规范。文档以现有后端接口为准，并参考京东页面在优惠券领取、账户优惠券、结算页用券和后台表格操作中的交互模式，作为 AI 辅助开发页面的技术依据。

## 3. 功能模块边界

| 模块 | 用户侧 | 职责边界 | 后端服务 |
| --- | --- | --- | --- |
| 用户认证 | 商家端 | 注册、登录、用户信息、修改密码、退出登录 | `merchant-admin` |
| 优惠券模板管理 | 商家端 | 创建模板、查询模板、结束模板、追加发行量 | `merchant-admin` |
| 推送任务管理 | 商家端 | 创建批量发券任务、查询任务、查看任务详情 | `merchant-admin` |
| 领券中心 | 用户端 | 查询券详情、展示可领券、同步/异步领取 | `engine` |
| 我的优惠券 | 用户端 | 展示已领取券、可用/不可用分组、跳转使用 | `settlement` |
| 预约提醒 | 用户端 | 创建、查询、取消开抢提醒 | `engine` |
| 优惠券结算 | 用户端/订单侧 | 结算页查券、选券、计算抵扣、提交用券 | `settlement`、`engine` |

## 5. 推荐技术栈

| 类别 | 推荐方案 | 说明 |
| --- | --- | --- |
| 框架 | Vue 3 + TypeScript | 适合表单、表格和状态驱动页面 |
| 构建 | Vite | 快速本地开发和模块化构建 |
| UI | Element Plus | 适合商家后台；用户端可二次封装京东风格组件 |
| 状态管理 | Pinia | 认证、券包、结算状态共享 |
| 请求 | Axios + `json-bigint` | 处理 Java `Long` 大整数精度 |
| 样式 | CSS Modules 或 SCSS | 支持主题 token 和模块化样式 |
| 测试 | Vitest + Testing Library + Playwright | 单元、组件和端到端测试 |

## 6. 京东参考说明

本文档参考京东公开页面常见交互模式，包括：

- 京东首页/频道页的红色品牌色、卡片分区、强主按钮和高密度商品信息展示。
- 京东领券中心的券卡片结构：面额突出、门槛说明、有效期、立即领取按钮、已领取态。
- 京东我的优惠券的标签分组：未使用、已使用、已过期，配合券状态弱化展示。
- 京东结算页的优惠券选择抽屉/弹层：展示最优券、不可用原因、确认使用。

差异设计：本项目既包含 C 端页面，也包含商家后台 B 端页面。商家端不照搬京东营销式布局，而采用更克制的后台工作台风格；用户端领券、券包、结算模块可强化京东式红色、券卡片和明确转化按钮。
