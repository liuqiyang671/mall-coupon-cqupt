## 1. 项目概述

邮惠券商城是一套基于微服务架构的优惠券管理系统，面向三类系统角色提供完整的优惠券生命周期管理能力。前端项目需根据角色类型呈现不同的界面与功能权限，实现"一套代码、多端适配、角色隔离"的设计目标。

**后端技术架构**：

- 微服务：Spring Cloud + Spring Boot 3.0.7
- 数据库：MySQL + ShardingSphere 分库分表
- 缓存：Redis + Redisson
- 消息队列：RocketMQ
- 认证：JWT + BCrypt 密码加密
- 网关：Spring Cloud Gateway

**前端核心要求**：

- 角色驱动的界面呈现（平台管理员 / 商家 / 普通用户）
- 统一的技术栈与组件库
- 响应式设计适配 PC 和移动端
- 高性能与良好的用户体验

## 2. 角色定义与功能模块

### 2.1 角色体系总览

系统基于 `roleType` 字段区分三种角色，数据库约束定义如下：

| roleType | 角色名称                  | 标识         | 登录入口              | 界面主题     |
| -------- | --------------------- | ---------- | ----------------- | -------- |
| 0        | 平台管理员（Platform Admin） | `admin`    | `/admin/login`    | 深色管理后台   |
| 1        | 商家（Merchant）          | `merchant` | `/merchant/login` | 浅色商家后台   |
| 2        | 普通用户（Regular User）    | `user`     | `/user/home`      | 明亮 C 端界面 |

**用户认证流程**：

```
用户访问 → 登录页 → 输入账号密码 → 后端验证 → 返回 JWT（含 roleType）
                                                    ↓
                                    根据 roleType 重定向到对应首页
                                    ├── roleType=0 → /admin/dashboard
                                    ├── roleType=1 → /merchant/template/list
                                    └── roleType=2 → /user/coupon-center
```

**顶部栏（Header）**：

- 支持折叠/展开（折叠后仅显示图标）
- 多级菜单（最多二级）
- 当前页面高亮显示
- 记住用户上次展开状态（localStorage）

- 顶部：面包屑导航
- 主体：页面内容
- 最小宽度 1000px，内容最大宽度 1400px 居中

### 3.3 页面跳转与交互流程

#### 3.3.1 登录跳转流程

```
访问任意页面
    │
    ├── 已登录
    │   ├── roleType=0 → /admin/dashboard
    │   ├── roleType=1 → /merchant/dashboard
    │   └── roleType=2 → /user/home
    │
    └── 未登录
        ├── 访问 /admin/* → 重定向 /admin/login?redirect=/admin/*
        ├── 访问 /merchant/* → 重定向 /merchant/login?redirect=/merchant/*
        └── 访问 /user/*（需登录页面）→ 重定向 /user/login?redirect=/user/*
```

#### 3.3.2 角色间隔离

| 规则                | 说明                                |
| ----------------- | --------------------------------- |
| Token 包含 roleType | JWT Payload 中携带 roleType          |
| 路由守卫校验            | 进入路由时校验 roleType 与路由前缀匹配          |
| 跨角色访问拦截           | 商家 Token 访问 /admin/\* 直接拒绝，返回 403 |
| 多角色登录             | 同一账号不同角色需分别登录（如用户同时是商家）           |

#### 3.3.3 核心交互流程

**商家创建优惠券流程**：

```

**用户结算流程**：

```

### 6.2 状态管理方案

#### 6.2.1 Auth Store（全局共享）

```typescript
// store/auth.ts
import { defineStore } from 'pinia'

interface AuthState {
  token: string
  userId: string
  username: string
  roleType: number | null  // 0=平台管理员 1=商家 2=普通用户
  shopNumber: number | null
  expireTime: number
}

export const useAuthStore = defineStore('auth', {
  state: (): AuthState => ({
    token: '',
    userId: '',
    username: '',
    roleType: null,
    shopNumber: null,
    expireTime: 0
  }),
  
  getters: {
    isLoggedIn: (state) => !!state.token && Date.now() < state.expireTime,
    isAdmin: (state) => state.roleType === 0,
    isMerchant: (state) => state.roleType === 1,
    isUser: (state) => state.roleType === 2
  },
  
  actions: {
    setAuth(data: { token: string; userId: string; username: string; roleType: number; shopNumber?: number; expireTime: number }) {
      this.token = data.token
      this.userId = data.userId
      this.username = data.username
      this.roleType = data.roleType
      this.shopNumber = data.shopNumber || null
      this.expireTime = data.expireTime
      this.persistAuth()
    },
    
    clearAuth() {
      this.token = ''
      this.userId = ''
      this.username = ''
      this.roleType = null
      this.shopNumber = null
      this.expireTime = 0
      localStorage.removeItem('auth')
      sessionStorage.removeItem('auth')
    },
    
    persistAuth() {
      const data = this.$state
      // 根据 remember 状态决定存储位置
      const storage = data.remember ? localStorage : sessionStorage
      storage.setItem('auth', JSON.stringify(data))
    },
    
    restoreAuth() {
      const saved = localStorage.getItem('auth') || sessionStorage.getItem('auth')
      if (saved) {
        const data = JSON.parse(saved)
        if (Date.now() < data.expireTime) {
          this.$state = data
          return true
        }
        this.clearAuth()
      }
      return false
    }
  }
})
```

        // ... 更多用户路由
      ]
    },
    { path: '/', redirect: '/user/home' },
    { path: '/:pathMatch(.*)*', component: () => import('@/views/NotFound.vue') }
  ]
})

// 全局路由守卫
router.beforeEach((to, _from, next) => {
  const authStore = useAuthStore()
  authStore.restoreAuth()
  
  if (to.meta.requiresAuth && !authStore.isLoggedIn) {
    next({ path: `/${to.path.split('/')[1]}/login`, query: { redirect: to.fullPath } })
    return
  }
  
  if (to.meta.guest && authStore.isLoggedIn) {
    const rolePrefix = authStore.isAdmin ? 'admin' : authStore.isMerchant ? 'merchant' : 'user'
    next(`/${rolePrefix}/dashboard`)
    return
  }
  
  if (to.meta.roleType !== undefined && authStore.roleType !== to.meta.roleType) {
    next('/403')
    return
  }
  
  next()
})

export default router
