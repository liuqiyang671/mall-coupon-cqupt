## 2. 现有功能模块优化方向

### 2.1 Gateway 网关模块

#### 2.1.1 路由配置修复

**问题**：路由 URI 包含路径部分，不符合 Spring Cloud Gateway 规范；缺少 `StripPrefix` 过滤器。

**当前配置**：
```yaml
uri: lb://oneCoupon-merchant-admin/api/merchant-admin/**
```

**修复方案**：
```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: oneCoupon-merchant-admin
          uri: lb://oneCoupon-merchant-admin
          predicates:
            - Path=/api/merchant-admin/**
          filters:
            - StripPrefix=1
```

#### 2.1.2 安全加固

| 问题 | 严重度 | 修复方案 |
|------|--------|---------|
| JWT 密钥硬编码 | 🔴 高 | 使用环境变量或 Vault/Nacos 加密配置注入 |
| 白名单 `startsWith` 过宽 | 🟡 中 | 改用精确匹配列表或 AntPathMatcher |
| 无限流熔断 | 🔴 高 | 集成 Sentinel 或 Resilience4j |
| 请求日志含敏感头 | 🟡 中 | 对 `Authorization` 头脱敏处理 |
| 无 Token 黑名单 | 🟡 中 | Redis 维护黑名单，用户改密/登出时加入 |
| 缺少 CORS 配置 | 🟡 中 | 添加全局 CORS 过滤器 |

#### 2.1.3 补充限流熔断

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: oneCoupon-engine
          uri: lb://oneCoupon-engine
          predicates:
            - Path=/api/engine/**
          filters:
            - name: RequestRateLimiter
              args:
                redis-rate-limiter.replenishRate: 100
                redis-rate-limiter.burstCapacity: 200
                key-resolver: "#{@userKeyResolver}"
            - name: CircuitBreaker
              args:
                name: engineCircuitBreaker
                fallbackUri: forward:/fallback
```

#### 2.1.4 补充 prod Profile 日志

当前 logback-spring.xml 仅配置 `dev` Profile，需补充 `prod` 配置：
- 日志级别调整为 WARN+
- 日志保留天数 30 天
- 敏感信息脱敏
- 接入日志采集系统（ELK/Loki）

### 2.5 Settlement 结算模块

#### 2.5.1 核心空实现修复（🔴 最高优先级）

**问题**：`CouponApplyServiceImpl.applySelectedCoupon()` 返回 null，整个优惠券应用流程断裂。

**修复方案**：

```java
@Override
public ApplyCouponRespDTO applySelectedCoupon(ApplyCouponReqDTO request) {
    UserCouponDO userCoupon = userCouponMapper.selectById(request.getCouponId());
    if (userCoupon == null || userCoupon.getStatus() != 0) {
        throw new ClientException("优惠券不可用");
    }

    CouponTemplateDO template = couponTemplateMapper.selectById(userCoupon.getCouponTemplateId());
    CouponTemplateDO typedCoupon = CouponFactory.createCoupon(template);

    CouponCalculationStrategy strategy = CouponFactory.getCouponCalculationStrategy(typedCoupon);
    BigDecimal discountAmount = strategy.calculate(request.getOrderAmount(), typedCoupon);

    BigDecimal payableAmount = request.getOrderAmount().subtract(discountAmount);
    if (payableAmount.compareTo(BigDecimal.ZERO) < 0) {
        payableAmount = BigDecimal.ZERO;
    }

    return ApplyCouponRespDTO.builder()
        .couponId(request.getCouponId())
        .orderAmount(request.getOrderAmount())
        .discountAmount(discountAmount)
        .payableAmount(payableAmount)
        .build();
}
```

#### 2.5.2 空接口清理

| 空接口 | 处理方式 |
|--------|---------|
| `SettlementGateway` | 实现为 Feign Client，调用 Engine 的结算单接口 |
| `SettlementHandler` | 实现为结算事件处理器，或删除 |
| `AsyncResponseHandler` | 实现为异步响应回调，或删除 |

#### 2.5.3 包名统一

**问题**：settlement 使用 `com.cqupt.settlement`，其他模块使用 `com.mall.cqupt`。

**修复**：重构包名为 `com.mall.cqupt.settlement`，涉及所有 Java 文件和配置。

**getCurrentUserId 修复示例**：

```java
private String getCurrentUserId() {
    ServletRequestAttributes attributes =
        (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
    if (attributes != null) {
        String userId = attributes.getRequest().getHeader("X-User-Id");
        if (StringUtils.hasLength(userId)) {
            return userId;
        }
    }
    throw new ClientException("用户未登录");
}
```

## 3. 需补充的核心商城功能清单

### 3.1 P0 级（系统闭环必需）

> 不实现这些功能，系统无法形成完整业务闭环。

| # | 功能 | 涉及模块 | 说明 | 工作量 |
|---|------|---------|------|--------|
| 1 | **优惠券应用计算** | Settlement | `applySelectedCoupon` 空实现，结算流程断裂 | 3天 |
| 2 | **搜索查询 API** | Search | ES 数据已同步但无法查询，搜索功能不可用 | 2天 |

#### 4.3.3 JSON 字段拆分建议

`t_coupon_template` 的 `receive_rule` 和 `consume_rule` 使用 JSON 类型，建议拆分高频查询字段：

```sql
ALTER TABLE t_coupon_template
  ADD COLUMN per_limit INT COMMENT '每人限领数量' AFTER stock,
  ADD COLUMN discount_value DECIMAL(10,2) COMMENT '优惠金额/折扣率' AFTER per_limit,
  ADD COLUMN min_purchase_amount DECIMAL(10,2) COMMENT '最低消费金额' AFTER discount_value,
  ADD COLUMN max_discount_amount DECIMAL(10,2) COMMENT '最大优惠金额' AFTER min_purchase_amount;
```

**收益**：避免 JSON 解析开销，支持索引查询和统计聚合。

## 6. 开发与维护注意事项及最佳实践

### 6.1 工程规范

#### 6.1.1 代码规范

| 规范项 | 要求 |

| 魔法值 | 禁止硬编码数字/字符串，使用枚举或常量类 |
| 方法长度 | 单方法不超过 80 行，超过则拆分 |
| 圈复杂度 | 单方法不超过 10，超过则拆分 |

#### 6.1.2 Git 规范

| 规范项 | 要求 |
|--------|------|
| 分支策略 | main（生产）/ develop（开发）/ feature/*（功能）/ hotfix/*（修复） |
| Commit 格式 | `type(scope): subject`，type: feat/fix/docs/style/refactor/test/chore |
| PR 要求 | 至少 1 人 Review + CI 通过方可合并 |
| .gitignore | 确保 .env、logs、target、node_modules 不入库 |

#### 6.1.3 API 设计规范

| 规范项 | 要求 |
|--------|------|
| URL 风格 | RESTful，小写 + 短横线，名词复数 |

| 响应格式 | 统一 `Result<T>` 封装，code="0" 成功 |
| 分页参数 | `current` + `size`，响应含 `total` + `records` |
| 错误响应 | 含 code + message + requestId |
