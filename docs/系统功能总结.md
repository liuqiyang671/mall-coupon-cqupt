系统采用 **Spring Cloud 微服务架构**，共 6 个业务微服务 + 1 个公共框架模块，通过 Gateway 统一对外暴露 API。前端基于 **Vue 3 + TypeScript** 构建，通过 Axios 调用后端微服务。

## 2. 角色体系

系统基于 `roleType` 字段定义了三种系统角色：

| 角色 | roleType | 说明 | 核心权限 |
|------|----------|------|---------|
| **平台人员** | 0 | 平台管理员/运营人员 | 查看全部商家数据、审核商家、管理商品分类和属性、系统级优惠券查看 |
| **商家** | 1 | 入驻商户 | 管理自身店铺商品、创建/管理优惠券模板、创建推送任务、经营数据统计 |
| **普通用户** | 2 | 终端消费者 | 领券中心领券、管理个人优惠券、购物车、结算用券、预约提醒 |

三角色共享统一的用户账号表（`t_user`），通过 JWT Token 中的 `roleType` 声明实现路由级和组件级权限隔离。

## 3. 后端微服务模块功能清单

### 3.1 Gateway 网关模块（端口 10000）

| 功能 | 说明 |
|------|------|
| **路由转发** | 5 条路由规则（merchant-admin / engine / settlement / distribution / search），基于 lb:// 负载均衡 |
| **JWT 认证** | TokenValidateFilter 拦截所有非白名单请求，验证 HMAC256 签名，提取用户信息通过 Header 透传 |
| **白名单** | `/api/merchant-admin/user/register`、`/api/merchant-admin/user/login`、Swagger 文档 |
| **请求日志** | UUID 流水号 + URI + 方法 + GET 参数 + 响应耗时，INFO/ERROR 按小时滚动 |

### 3.2 Framework 公共框架模块

| 功能 | 说明 |
|------|------|
| **统一异常体系** | AbstractException 基类 → ClientException(A) / ServiceException(B) / RemoteException(C) |
| **全局异常处理** | GlobalExceptionHandler 拦截参数校验异常 / 业务异常 / 兜底异常 |
| **统一返回格式** | `Result<T>` 封装：code("0"=成功)、message、data、requestId |
| **防重复提交** | `@NoDuplicateSubmit` 注解 + Redisson 分布式锁（路径+userId+参数MD5） |
| **MQ 幂等消费** | `@NoMQDuplicateConsume` 注解 + Redis SETNX + SpEL 表达式 Key |
| **缓存配置** | 自定义 Redis Key 序列化器，支持全局前缀隔离 |

### 3.5 Settlement 结算模块（端口 10030）

#### 优惠券应用（1 个接口）

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/settlement/apply-coupon/{couponId}` | 应用优惠券折扣订单金额 |

#### 优惠券查询（2 个接口）

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/settlement/coupon-query` | 查询用户可用/不可用优惠券（多线程版，CompletableFuture 并行） |
| POST | `/api/settlement/coupon-query-sync` | 同步查询用户可用/不可用优惠券（单线程版） |

#### 计算策略（3 种）

| 策略类 | 优惠券类型 | 计算逻辑 |
|--------|-----------|---------|
| FixedDiscountCalculationStrategy | 立减券 | 直接返回固定折扣金额 |
| ThresholdCalculationStrategy | 满减券 | 金额 >= 门槛返回折扣金额，否则返回 0 |
| DiscountCalculationStrategy | 折扣券 | 订单金额 × 折扣率 |

### 3.6 Distribution 分发模块（端口 10040）

纯 MQ 消费者驱动模块，无 REST API：

| 消费者 | 功能 |
|--------|------|
| CouponTaskSendExecuteConsumer | 延迟执行 Topic，定时任务到期后转发执行命令 |
| CouponTaskExecuteConsumer | 任务执行 Topic，读取 Excel、逐行投递分发消息 |
| CouponExecuteDistributionConsumer | 分发 Topic，批量扣减库存 + 批量插入用户券 + 冲突自动排查递归重试 |
| CouponExecuteSendMessageConsumer | 消息通知 Topic，发送通知（站内信/短信/邮件等） |

#### 消息发送服务（5 种策略）

| 策略 | 通知方式 |
|------|---------|
| SiteMessageSendService | 站内信 |
| SMSMessageSendServiceImpl | 短信 |
| MailMessageSendServiceImpl | 邮件 |
| WeChatMessageSendServiceImpl | 微信 |
| ApplicationMessageSendServiceImpl | 应用弹框 |

### 3.7 Search 搜索模块（端口 10050）

## 7. 核心业务链路

### 7.1 优惠券模板创建与发布

```
商家/平台 → Merchant-Admin 创建模板 → 责任链验证
    → 写入 DB（ShardingSphere 分片） → Redis Hash 预热缓存
    → 操作日志记录（@LogRecord）
```

### 7.2 用户领券流程（同步模式）

```
用户请求领券 → Engine 查询模板缓存（Redis → 布隆过滤器 → 分布式锁 → DB）
    → 校验时间/库存/限领 → Lua 脚本原子扣减库存 + 限领次数
    → 编程式事务：DB 扣库存 + 插入用户券 + Redis ZSet 缓存
    → 写后查询验证（防止主从复制数据丢失）
```

### 7.3 用户领券流程（MQ 异步模式）

```
用户请求领券 → Lua 脚本原子扣减库存 + 限领次数
    → 投递 RocketMQ 消息 → 返回领券成功
    → Distribution 模块消费消息 → 异步落库
```

### 7.4 批量发券流程

```
商家创建推送任务 → Merchant-Admin 异步统计 Excel 行数
    → Redisson 延迟队列兜底 → 投递 MQ 延迟消息
    → Distribution 延迟到期消费 → 读取 Excel 逐行解析
    → 逐行投递分发消息 → 批量扣减库存 + 批量插入用户券
    → 冲突自动排查（递归重试） → 更新任务状态
    → 分发完成投递消息通知
```

### 7.5 预约提醒流程

```
用户创建预约 → Engine 位图存储提醒信息（Long 型 Bitmap）
    → 投递 MQ 延迟消息 → 到期触发提醒发送（站内信/短信/邮件）
```

### 7.6 优惠券结算流程

```
用户选择优惠券 → Settlement 查询可用券（Redis Pipeline + CompletableFuture 并行）
    → Engine 创建结算单（分布式锁 + 折扣计算 + 锁定优惠券）
    → 用户支付 → 核销结算单（结算单已支付 + 优惠券已使用）
    → 用户退款 → 退款结算单（结算单已退款 + 优惠券退回未使用）
```

### 7.7 ES 搜索数据同步流程

```
MySQL 变更（INSERT/UPDATE/DELETE）
    → Canal 监听 Binlog → 投递 MQ
    → Search 模块消费消息 → 同步到 Elasticsearch
    → 优惠券模板状态变更自动反映到搜索结果
```

### 7.8 购物车业务流程

```
用户添加商品 → Engine 校验登录状态 → 同商品已存在则累加数量（上限 999）
    → 否则新增购物车项（最多 50 种商品） → 写入 DB（ShardingSphere 按 user_id 分片）
    → 清除 Redis 缓存
用户查看购物车 → 查询 DB 购物车项 → 关联商品信息（从商品缓存读取）
    → 计算汇总数据（总金额/已选金额/节省金额/税费 6%/应付金额）
```

## 9. 安全机制

| 安全措施 | 实现位置 | 说明 |
|----------|---------|------|
| **JWT 认证** | Gateway | HMAC256 签名、Token 解析透传用户信息（X-User-Id / X-Username / X-Shop-Number） |
| **Token 会话管理** | Merchant-Admin | Redis 存储 Token、修改密码/退出删除 Token |
| **登录防暴力破解** | Merchant-Admin | Redis 失败计数 + 5 次失败锁定 30 分钟 |
| **密码加密** | Merchant-Admin | BCryptPasswordEncoder |
| **横向越权防护** | Merchant-Admin | shopNumber + source + id 三重条件校验 |
| **角色权限控制** | 全模块 | 平台/商家/普通用户三角色、路由级 + 组件级双重权限守卫 |
| **防重复提交** | Framework | Redisson 分布式锁 + 请求路径 + 参数 MD5 |
| **MQ 幂等消费** | Framework | Redis SETNX + SpEL Key + 失败回滚 |
| **布隆过滤器** | Engine | 防缓存穿透（模板查询 + 取消提醒判断） |
| **分布式锁** | Engine | Redisson 锁（缓存重建、结算单操作） |
| **Lua 原子操作** | Engine | 库存扣减 + 限领次数原子执行 |
| **乐观锁** | 多模块 | 库存增减（WHERE stock >= quantity）、更新提醒（WHERE information = ?） |
| **操作日志** | Merchant-Admin | @LogRecord（mzt-bizlog-sdk）记录模板创建/结束/增发量 |
| **请求日志** | Gateway | UUID 流水号 + 请求信息 + 响应耗时 |
